# Gomoku-Demo 五子棋项目计划

## 一、项目概述

**项目名称**: Gomoku-Demo
**项目类型**: 纯前端 HTML + Canvas 游戏
**项目目标**: 实现一个功能完整、界面美观的网页版五子棋游戏
**运行环境**: 现代浏览器（Chrome / Firefox / Safari / Edge）

---

## 二、功能需求

### 2.1 核心游戏功能
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 棋盘绘制 | 使用 Canvas 绘制 15×15 标准五子棋棋盘 | P0 |
| 落子功能 | 点击棋盘交叉点落子，黑白交替 | P0 |
| 胜负判断 | 横/竖/斜四个方向检测五子连珠 | P0 |
| 游戏重置 | 点击"重新开始"按钮清空棋盘 | P0 |
| 悔棋功能 | 撤销上一步落子操作 | P1 |
| 落子提示 | 标记最后一颗落子位置 | P1 |

### 2.2 对战模式
| 模式 | 描述 | 优先级 |
|------|------|--------|
| 双人对战 | 两名玩家在同一设备上交替落子 | P0 |
| 人机对战 | 玩家对战 AI（简单策略） | P1 |

### 2.3 界面功能
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 当前回合显示 | 显示当前轮到哪方落子 | P0 |
| 胜利弹窗 | 游戏结束时显示获胜方提示 | P0 |
| 棋子动画 | 落子时添加简单动画效果 | P2 |
| 响应式布局 | 适配不同屏幕尺寸 | P1 |

## 三、技术方案

### 3.1 技术栈
| 技术 | 用途 |
|------|------|
| HTML5 | 页面结构 |
| Canvas 2D API | 棋盘和棋子绘制 |
| CSS3 | 页面样式、动画 |
| 原生 JavaScript (ES6+) | 游戏逻辑 |

### 3.2 文件结构

```
gomoku-demo/
├── index.html          # 主页面

├── css/

    ├── board.js        # 棋盘绘制模块

    ├── ai.js           # AI 对战模块
    └── main.js         # 入口，事件绑定
```

### 3.3 Canvas 坐标系设计
```
棋盘参数:
- 格子数: 15 × 15
- 格子大小: 40px
- 棋盘边距: 30px
- Canvas 总宽高: 15 × 40 + 2 × 30 = 660px
- 交叉点坐标: x = 边距 + 列索引 × 格子大小
              y = 边距 + 行索引 × 格子大小
```

### 4.2 胜负判断算法
```
对每个落子位置，检查4个方向：
  - 水平方向: (0, +1)
  - 垂直方向: (+1, 0)
  - 正斜线:   (+1, +1)
  - 反斜线:   (+1, -1)

每个方向统计连续同色棋子数量 >= 5 则判定获胜
```

### 4.3 AI 策略（简单版）
```
评分规则（启发式搜索）:
  - 五连: 100000 分（立即获胜）
  - 活四: 10000 分
  - 冲四: 1000 分
  - 活三: 500 分
  - 眠三: 100 分
  - 活二: 50 分

AI 决策：遍历空位，取我方得分和对方威胁分的最大值
```

## 七、验收标准

1. 棋盘正确渲染 15×15 格子，星位标记准确
2. 黑白双方交替落子，已落子位置不可重复
3. 五子连珠后正确判胜并停止游戏
4. 悔棋、重置功能正常工作
5. 界面美观，在主流浏览器中无兼容问题
6. AI 模式下能做出合理落子（不随机乱走）
